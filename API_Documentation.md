# Zorvyn Finance API Documentation

This document outlines the available endpoints, required authentication mechanisms, and request formatting for the Zorvyn Finance backend application. The system utilizes stateless JSON Web Tokens (JWT) for session management and enforces role-based access control (RBAC).

---

## 1. Authentication

To interact with the financial endpoints, a valid Bearer token is required in the `Authorization` header. First, you must provision an account to receive this token.

### 1.1 Register Administrator Account
Administrative privileges are necessary to create, update, or delete records.

*   **Endpoint:** `POST /api/auth/register`
*   **Authorization:** None
*   **Content-Type:** `application/json`
*   **Payload:**
    ```json
    {
        "username": "adminUser",
        "email": "admin@zorvyn.com",
        "password": "securepassword",
        "role": "ADMIN"          
    }
    ```
*Response Protocol:* The system will return a 200 OK containing a JWT `token`. Store this token.

### 1.2 Register Viewer Account (Role Validation Testing)
To verify the RBAC constraints, you can provision a standard read-only account.

*   **Endpoint:** `POST /api/auth/register`
*   **Payload:** Use the identical schema above, but assign `"role": "VIEWER"`.

## 2. Financial Record Management

All requests in this module require the Authorization header format: `Bearer <token>`

### 2.1 Create Financial Record
*   **Endpoint:** `POST /api/records`
*   **Role Required:** `ADMIN`
*   **Payload:**
    ```json
    {
        "amount": 5000.00,
        "type": "INCOME",
        "category": "Salary",
        "date": "2026-04-05",
        "notes": "Initial deposit"
    }
    ```
*Note: Supplying a negative amount or omitting fields will trigger a 400 Bad Request exception generated by the validation interceptor.*

### 2.2 Retrieve Records
*   **Endpoint:** `GET /api/records`
*   **Role Required:** `ADMIN`, `ANALYST`, or `VIEWER`
*   **Payload:** None required. Returns a JSON array of all registered records.

### 2.3 Retrieve Specific Record by ID
*   **Endpoint:** `GET /api/records/{id}` (e.g., `/api/records/1`)
*   **Role Required:** `ADMIN`, `ANALYST`, or `VIEWER`

### 2.4 Update Record
*   **Endpoint:** `PUT /api/records/{id}`
*   **Role Required:** `ADMIN`
*   **Payload:** Requires the same JSON schema used in **2.1**.

### 2.5 Delete Record
*   **Endpoint:** `DELETE /api/records/{id}`
*   **Role Required:** `ADMIN`
*   **Payload:** None required.
*Note: A successful deletion returns a standard 204 No Content. Attempting this with a VIEWER token will correctly return a 403 Forbidden payload.*

## 3. Analytics & Dashboard

### 3.1 Fetch Dashboard Aggregate Summary
*   **Endpoint:** `GET /api/dashboard/summary`
*   **Role Required:** Any authenticated role
*   **Response:** Aggregated calculations executed at the persistence layer, returning `totalIncome`, `totalExpense`, and `netBalance`.
