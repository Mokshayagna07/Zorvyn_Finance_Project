=================================================
 ZORVYN FINANCE API - TESTING GUIDE (POSTMAN)
=================================================
Base URL: http://localhost:8080

IMPORTANT: 
For all endpoints below (except Auth), you must include the JWT Token in the HTTP Headers.
Header Key:   Authorization
Header Value: Bearer <your_token_here>


-------------------------------------------------
 1. AUTHENTICATION (No Token Required)
-------------------------------------------------

[REGISTER AN ACCOUNT]
Method: POST
URL:    http://localhost:8080/api/auth/register
Body (RAW JSON):
{
    "username": "adminUser",
    "email": "admin@test.com",
    "password": "password123",
    "role": "ADMIN"          // Can be "ADMIN", "ANALYST", or "VIEWER"
}


[LOGIN TO ACCOUNT]
Method: POST
URL:    http://localhost:8080/api/auth/login
Body (RAW JSON):
{
    "email": "admin@test.com",
    "password": "password123"
}
Response: Copy the "token" value! You will need it for the next requests.


-------------------------------------------------
 2. DASHBOARD (All Roles Allowed)
-------------------------------------------------

[GET DASHBOARD SUMMARY]
Method: GET
URL:    http://localhost:8080/api/dashboard/summary
Header: Authorization: Bearer <token>
Response: Returns Total Income, Total Expense, and Net Balance.


-------------------------------------------------
 3. FINANCIAL RECORDS (Strict Role Verification)
-------------------------------------------------

[CREATE NEW RECORD] (Action: ADMIN Only)
Method: POST
URL:    http://localhost:8080/api/records
Header: Authorization: Bearer <token>
Body (RAW JSON):
{
    "amount": 2500.50,
    "type": "INCOME",        // Must be "INCOME" or "EXPENSE"
    "category": "Salary",
    "date": "2026-04-04",
    "notes": "April Paycheck"
}


[GET ALL RECORDS] (Action: ADMIN or ANALYST Only)
Method: GET
URL:    http://localhost:8080/api/records
Header: Authorization: Bearer <token>


[GET A SINGLE RECORD BY ID] (Action: ADMIN or ANALYST Only)
Method: GET
URL:    http://localhost:8080/api/records/1
Header: Authorization: Bearer <token>


[UPDATE RECORD BY ID] (Action: ADMIN Only)
Method: PUT
URL:    http://localhost:8080/api/records/1
Header: Authorization: Bearer <token>
Body (RAW JSON):
{
    "amount": 3000.00,
    "type": "INCOME",
    "category": "Salary Bonus",
    "date": "2026-04-05",
    "notes": "Updated to include bonus"
}


[DELETE RECORD BY ID] (Action: ADMIN Only)
Method: DELETE
URL:    http://localhost:8080/api/records/1
Header: Authorization: Bearer <token>


-------------------------------------------------
 HOW TO TEST VALIDATIONS & ACCESS CONTROL
-------------------------------------------------
1. Try sending the [CREATE NEW RECORD] JSON without an "amount". You will see a 400 Bad Request JSON generated by the GlobalExceptionHandler.
2. Login as a VIEWER and try hitting [GET ALL RECORDS]. You will see a 403 Forbidden AccessDenied error because of your `@PreAuthorize` Backend validations!
