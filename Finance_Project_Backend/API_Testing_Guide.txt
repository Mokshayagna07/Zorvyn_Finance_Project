=================================================
 ZORVYN FINANCE API - STEP-BY-STEP TESTING GUIDE
=================================================

This guide will walk you through strictly testing every single endpoint in Postman from start to finish.

-------------------------------------------------
 STEP 1: CREATE YOUR ADMIN ACCOUNT
-------------------------------------------------
[CREATE ADMIN USER]
• Method: POST
• URL: http://localhost:8080/api/auth/register
• Token Needed: NO
• Body (Raw -> JSON):
{
    "username": "AdminBoss",
    "email": "admin@zorvyn.com",
    "password": "password123",
    "role": "ADMIN"          
}
🟢 What to do next: Copy the long "token" string from the response! You will need this token for actions that only an ADMIN can do.

-------------------------------------------------
 STEP 2: CREATE A VIEWER ACCOUNT (For Testing)
-------------------------------------------------
[CREATE VIEWER USER]
• Method: POST
• URL: http://localhost:8080/api/auth/register
• Token Needed: NO
• Body (Raw -> JSON):
{
    "username": "ViewerGuest",
    "email": "viewer@zorvyn.com",
    "password": "password123",
    "role": "VIEWER"          
}
🟢 What to do next: Copy this "token" and save it somewhere else as your Viewer Token.

-------------------------------------------------
 STEP 3: CREATE A FINANCIAL RECORD (Admin Only!)
-------------------------------------------------
[ADD RECORD]
• Method: POST
• URL: http://localhost:8080/api/records
• Token Needed: YES 👉 Use your ADMIN token from Step 1.
   (Go To Authorization Tab -> Select "Bearer Token" -> Paste Admin Token)
• Body (Raw -> JSON):
{
    "amount": 5000.00,
    "type": "INCOME",
    "category": "Salary",
    "date": "2026-04-05",
    "notes": "First paycheck"
}
*Note: If you omit the amount or specify a negative amount, you will get a 400 Bad Request error (Validation check!)*
🟢 What to do next: Look at the response and copy the "id" of the record you just created (e.g., id: 1).

-------------------------------------------------
 STEP 4: VIEW ALL RECORDS (Anyone can view)
-------------------------------------------------
[GET RECORDS]
• Method: GET
• URL: http://localhost:8080/api/records
• Token Needed: YES 👉 You can use EITHER the Admin or Viewer Token here.
• Body: NONE (Leave it empty)

-------------------------------------------------
 STEP 5: VIEW A SINGLE RECORD
-------------------------------------------------
[GET SINGLE RECORD BY ID]
• Method: GET
• URL: http://localhost:8080/api/records/1   <-- Change '1' to your actual record ID
• Token Needed: YES 👉 You can use EITHER the Admin or Viewer Token here.
• Body: NONE (Leave it empty)

-------------------------------------------------
 STEP 6: UPDATE A RECORD (Admin Only!)
-------------------------------------------------
[UPDATE RECORD]
• Method: PUT
• URL: http://localhost:8080/api/records/1   <-- Change '1' to your actual record ID
• Token Needed: YES 👉 MUST use ADMIN token. (If you use Viewer, you get 403 Forbidden).
• Body (Raw -> JSON):
{
    "amount": 6000.00,
    "type": "INCOME",
    "category": "Salary Bonus",
    "date": "2026-04-05",
    "notes": "Updated salary with a bonus!"
}

-------------------------------------------------
 STEP 7: VIEW THE DASHBOARD SUMMARY
-------------------------------------------------
[GET DASHBOARD]
• Method: GET
• URL: http://localhost:8080/api/dashboard/summary
• Token Needed: YES 👉 You can use EITHER the Admin or Viewer token.
• Body: NONE
🟢 What happens: It will calculate your total income vs expenses and give you the Net Balance perfectly!

-------------------------------------------------
 STEP 8: DELETE A RECORD (Admin Only!)
-------------------------------------------------
[DELETE RECORD]
• Method: DELETE
• URL: http://localhost:8080/api/records/1   <-- Change '1' to your actual record ID
• Token Needed: YES 👉 MUST use ADMIN token.
• Body: NONE
🟢 What happens: You will get an empty success response showing the record has been permanently deleted from your MySQL database. If you try to do this with your Viewer token, you will receive a 403 Forbidden error!
